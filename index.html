Credenciales de Supabase — se guardan solo en este navegador

🔐 Conexión Supabase

Ingresá tus credenciales de Supabase. Se almacenan en localStorage del navegador — nunca se envían a ningún servidor externo. Para producción en Vercel, podés además declarar SUPABASE_URL y SUPABASE_KEY como variables de entorno (ver instrucciones abajo).

📋 Variables de entorno en Vercel (producción)

Si desplegás en Vercel y querés que las credenciales no estén en el código:

  1. Ir a tu proyecto en Vercel → Settings → Environment Variables
  2. Agregar SUPABASE_URL con el valor de tu URL
  3. Agregar SUPABASE_KEY con tu Anon Key
  4. Como este es un HTML estático, las vars de entorno de Vercel no se inyectan automáticamente en el HTML cliente. La forma más segura es usar el formulario de configuración de esta página (localStorage) o bien crear un pequeño api/config.js como endpoint serverless en Vercel que devuelva las credenciales a la app.

Recomendación para app personal: usar localStorage (formulario de arriba) es suficiente y sencillo. La Anon Key de Supabase está diseñada para ser pública — la seguridad real la dan las Row Level Security (RLS) policies en Supabase.
